1661 /*
1662  * Queue sets provide a mechanism to allow a task to block (pend) on a read
1663  * operation from multiple queues or semaphores simultaneously.
1664  *
1665  * See FreeRTOS/Source/Demo/Common/Minimal/QueueSet.c for an example using this
1666  * function.
1667  *
1668  * A queue set must be explicitly created using a call to xQueueCreateSet()
1669  * before it can be used. Once created, standard FreeRTOS queues and semaphores
1670  * can be added to the set using calls to xQueueAddToSet().
1671  * xQueueSelectFromSet() is then used to determine which, if any, of the queues
1672  * or semaphores contained in the set is in a state where a queue read or
1673  * semaphore take operation would be successful.
1674  *
1675  * Note 1: See the documentation on http://wwwFreeRTOS.org/RTOS-queue-sets.html
1676  * for reasons why queue sets are very rarely needed in practice as there are
1677  * simpler methods of blocking on multiple objects.
1678  *
1679  * Note 2: Blocking on a queue set that contains a mutex will not cause the
1680  * mutex holder to inherit the priority of the blocked task.
1681  *
1682  * Note 3: An additional 4 bytes of RAM is required for each space in a every
1683  * queue added to a queue set. Therefore counting semaphores that have a high
1684  * maximum count value should not be added to a queue set.
1685  *
1686  * Note 4: A receive (in the case of a queue) or take (in the case of a
1687  * semaphore) operation must not be performed on a member of a queue set unless
1688  * a call to xQueueSelectFromSet() has first returned a handle to that set member.
1689  *
1690  * @param uxEventQueueLength Queue sets store events that occur on
1691  * the queues and semaphores contained in the set. uxEventQueueLength specifies
1692  * the maximum number of events that can be queued at once. To be absolutely
1693  * certain that events are not lost uxEventQueueLength should be set to the
1694  * total sum of the length of the queues added to the set, where binary
1695  * semaphores and mutexes have a length of 1, and counting semaphores have a
1696  * length set by their maximum count value. Examples:
1697  * + If a queue set is to hold a queue of length 5, another queue of length 12,
1698  * and a binary semaphore, then uxEventQueueLength should be set to
1699  * (5 + 12 + 1), or 18.
1700  * + If a queue set is to hold three binary semaphores then uxEventQueueLength
1701  * should be set to (1 + 1 + 1 ), or 3.
1702  * + If a queue set is to hold a counting semaphore that has a maximum count of
1703  * 5, and a counting semaphore that has a maximum count of 3, then
1704  * uxEventQueueLength should be set to (5 + 3), or 8.
1705  *
1706  * @return If the queue set is created successfully then a handle to the created
1707  * queue set is returned. Otherwise NULL is returned.
1708  */
1709 QueueSetHandle_t xQueueCreateSet(const UBaseType_t uxEventQueueLength) PRIVILEGED_FUNCTION;
